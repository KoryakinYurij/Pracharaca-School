# CONTENT_GUIDE.md

Руководство по добавлению контента для Noble Notes.

## 1. Файлы контента

- Тема: `content/topics/<topic-slug>/meta.json`
- Уроки: `content/topics/<topic-slug>/lessons/<order>-<slug>.json`

Рекомендуемая нотация имени урока:
- `01-noble-basics.json`
- `02-qa-craft.json`

## 2. Схема `meta.json`

```json
{
  "slug": "krasivyi-konspekt",
  "order": 1,
  "title": "Красивый конспект",
  "subtitle": "Локальный формат обучения",
  "description": "Краткое описание темы"
}
```

## 3. Схема урока

```json
{
  "slug": "qa-craft",
  "order": 2,
  "title": "Композиция Q/A карточек",
  "intro": "Короткое вступление (опционально)",
  "cards": [
    {
      "q": "Вопрос?",
      "a": [
        { "kind": "summary", "body": "1-2 предложения" },
        { "kind": "definition", "pairs": [{ "term": "...", "desc": "..." }] },
        { "kind": "steps", "items": ["Шаг 1", "Шаг 2"] },
        {
          "kind": "compare",
          "table": {
            "headers": ["Подход", "Когда применять"],
            "rows": [["A", "..."], ["B", "..."]]
          }
        }
      ]
    }
  ]
}
```

## 4. Поля секции ответа

- `kind` — тип секции.
- `title` (опционально) — заголовок секции.
- `body` (опционально) — обычный текст с переносами строк.
- `items` (опционально) — массив строк для списков.
- `pairs` (опционально) — массив пар `{ "term": string, "desc": string }`.
- `columns` (опционально) — массив колонок `{ "title": string, "items": string[] }`.
- `table` (опционально) — объект таблицы `{ "headers": string[], "rows": string[][] }`.

## 4a. Опциональные поля секции

Помимо основных полей, секция может содержать:

- `variant` (опционально) — вариант рендеринга.
- `visualHint` (опционально) — визуальная подсказка.
- `author` (опционально) — автор секции.
- `migration` (опционально) — метаданные миграции: `{ target, rendererOrder, isKnownKind, deprecationStatus, legacyPattern, wave, note }`.

Рендерер использует политику `new -> legacy`: сначала новый рендер, затем legacy. Для неизвестных kind используется safe fallback в виде простого текста.

## 5. Поддерживаемые `kind`

Старые:
- `summary` — короткая выжимка мысли
- `details` — основной развёрнутый ответ
- `steps` — пошаговый список; лучше через `items`, но `body` с переносами тоже поддерживается
- `example` — конкретный сценарий или мини-кейс
- `note` — мягкая уточняющая заметка
- `warning` — важное ограничение или типичная ошибка
- `quote` — короткая формула-принцип

Новые:
- `definition` — список терминов; используйте `pairs[]`
- `checklist` — чеклист; используйте `items[]`
- `pitfalls` — частые ошибки; используйте `items[]` или `body`
- `dosdonts` — формат «делай / не делай»; используйте `columns[]`
- `compare` — сравнение; используйте `columns[]` или `table`
- `mnemonic` — запоминалка; используйте `body`
- `references` — источники; используйте `items[]`

## 6. Правила композиции ответа

- 2-5 секций на карточку (редко 6).
- Начинайте с `summary` (1-2 предложения).
- Затем добавляйте 1 структурную секцию: `steps`, `checklist`, `compare`, `dosdonts` или `definition`.
- После структуры добавляйте `example` или `pitfalls`.
- `warning` используйте редко, только когда риск действительно критичен.
- `quote` — максимум 1 на карточку.
- Не более 1-2 «украшений» на карточку, избегайте перегруза.

## 7. Рекомендации по стилю "noble"

- Тон благородный и лаконичный.
- Без жаргона и без «крикливых» emoji.
- Формулируйте мысли спокойно и предметно.
- Каждая секция должна добавлять новый смысловой слой, а не повторять предыдущую.

## 8. Проверка перед сохранением

- JSON валиден (нет лишних запятых и сломанных кавычек).
- `slug` уникален во всех уроках.
- `order` не дублируется.
- У карточки есть `q` и минимум одна секция в `a`.
- Для структурных `kind` заполнено соответствующее поле (`items`, `pairs`, `columns`, `table`).
- Текст только на русском.
